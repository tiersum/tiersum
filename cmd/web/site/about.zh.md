# 关于 TierSum

## 产品概述

TierSum 是一个**分层摘要知识库**——基于多层抽象和热/冷文档分层的无需 RAG 的文档检索系统。

## 为何选择 TierSum

常见检索会把长文切成大量小块再做相似度匹配，容易**丢失结构与语境**。TierSum 用**分层摘要**（文档 → 章节 → 原文）保留知识结构，并配合**标签与主题**做可解释的导航与筛选，而不是只靠向量「猜」片段边界。

在**热路径**上，**入库时**即可由 AI 写好标签、文档概述与章节级提要，形成后续检索依赖的**「已铺好的」语义层**；**渐进式查询**在此基础上沿 **标签 → 文档 → 章节** 逐步收窄，每一步由大模型打分，像先扫目录再翻到对应小节，路径**可解释、可复查**。**冷路径**则省去大部分入库时 LLM 成本，仍以**整章**建索引与返回；访问频繁时可**晋升**为热文档，获得同样的预摘要能力。

> **冷热一致：以章节为单元。** 无论走「热」还是「冷」，系统都以 **Markdown 章节**（按标题划分的自然段）作为工作与检索粒度，而不是固定长度的随机碎片。摘要、渐进式收窄与冷索引检索都沿这一边界展开，**保证语义从头到尾完整、可理解**。

## 能做什么

- **搜索** — 用自然语言提问。**渐进式查询**沿 **标签 → 文档 → 章节** 推进，如同先扫提纲再点开章节：每一步在已有 **预摘要与预标签**（热路径文档）之上做相关性判断，必要时再汇总为带引用的回答（视配置而定）。
- **文档** — 导入 Markdown 等；热路径按 **章节** 生成摘要与标签；冷路径同样按 **章节** 建索引与返回命中，冷热都以章节为边界，避免正文被无意义切碎。
- **标签** — 浏览全库标签，并按 **主题** 分组浏览；「重归类」会用 LLM 根据当前标签目录刷新主题，便于随库增长维护导航。

## 热文档与冷文档（通俗理解）

差别主要在**入库时做多少 LLM 分析**，而不是把知识切成不同大小的「块」。**热与冷都以章节为中心：**检索、排序与展示都建立在 **完整 Markdown 章节** 上，沿标题结构保持语义边界，避免任意碎片破坏语境。

- **热文档**即上文「入库时预写摘要与标签」的路径；渐进式查询主要依托这一层展开，通常受**每小时配额**约束以控制成本。
- **冷文档**入库时几乎不做完整 LLM 分析，但同样按 **章节** 建索引，用 **BM25 + 向量** 混合检索，命中返回**整章正文**；查询次数达到阈值后可自动向「热」晋升。

## 适合谁

以 Markdown 为中心的知识团队：内部手册、架构说明、运维笔记、研究纪要，以及需要 **REST / MCP** 对接的智能体与工作流。浏览器界面与程序化接口共用同一套数据，减少「人看的知识库」和「机器用的接口」分裂。
